# Algorithm

Writing an algorithm is about expressing a solution clearly, step by step, in a way that can later be converted into code.

"Think of it as a bridge between problem understanding and coding."

## Standard Algorithm Template

    Algorithm <Algorithm_Name>(Input)

    1. Start

    2. Define input(s)
       - What values are given?

    3. Define output(s)
       - What should be returned?

    4. Initialize variables
       - Counters, flags, data structures, etc.

    5. Process / Logic
       - Step-by-step operations
       - Use loops, conditions, formulas

    6. Return result

    7. End

## Interview-Friendly Template

    Algorithm \<Name\>(Input):

    Input: Description of input
    Output: Description of output

    Begin
        Step 1: Initialize required variables

        Step 2: Handle edge cases
                (e.g., empty input, n = 0, etc.)

        Step 3: Apply core logic
                Loop / recursion / condition

        Step 4: Store/update result

        Step 5: Return result
    End

## How to Think While Writing Algorithm

    1. Understand the problem
        - What is given?
        - What is required?

    2. Identify approach
        - Brute force?
        - Optimized?
        - Data structures needed?

    3. Break into steps
        - Input → Process → Output

    4. Write in simple English first

  Then refine into structured steps.

## Common Patterns to Practice

Most DSA algorithms follow these patterns:

- Loop-based
- Two pointers
- Sliding window
- Recursion
- Divide & conquer
- Greedy
- Dynamic programming
